## Why this file exists

The IntentPacket is the versioned contract between the onboard brain (Hailo-10H LLM state machine) and landside consumers. **The schema is owned by the brain project.** This file exists in the landside workspace as a navigation pointer — read the source-of-truth file for the actual schema.

Landside consumers of IntentPacket:
- Depot Briefing Interface — surfaces packets with `request_landside_tasking=True` and groups by `agent_state`/`severity`
- Fleet Manager AI Interface — stores in PostgreSQL + pgvector (embeds `incident_summary`); uses `packet_id` as idempotency key on insert
- Rail MCP Server — `get_intent_packet` and `search_fleet_events` tools read from `intent_packets` table

## Landside contract obligations

Landside MUST:
- Deduplicate on `packet_id` (idempotent insert)
- Discard display state after `expires_at` — never resurface stale CRITICAL alerts after reconnect
- Surface `confidence_score` and render low-confidence (<0.75) packets distinctly
- Cite `source_signals` when displaying packet content to a user
- Treat schema as untrusted external input — validate before any AI decision path

## When the brain bumps the schema

Brain owns versioning. A `schema_version` bump triggers a landside migration story. Backward compatibility for currently-supported alert types is a brain-side concern; landside must surface unknown `agent_state` values as raw events rather than silently discarding them.
