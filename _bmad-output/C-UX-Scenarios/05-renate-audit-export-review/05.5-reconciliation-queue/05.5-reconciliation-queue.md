## Overview

**Page Purpose:** The reconciliation queue is where paper-vs-shadow divergences (Renate's central concern) are surfaced, triaged, and resolved. Each row is a paper sign-off that was compared against its corresponding shadow row by the reconciler (manual or automated); rows where the comparison flagged a difference appear here as divergences. Each row carries the reason code, resolution status, IdP identity of the resolver, and Phase 2 readiness impact indicator.

**User Situation:** Renate clicks "Open reconciliation queue" from 05.1 or 05.2. She wants the pilot-to-date view: how many sign-offs, what fraction matched, how many diverged, how many remain unresolved.

**Success Criteria:**
- Composite stats visible at-glance (total sign-offs, match rate, pending count, divergence count)
- Each divergence has a reason code, resolution status, and IdP-identified resolver
- Persistent unresolved divergences > 24h are visually surfaced (these affect Phase 2 readiness composite)
- Filter by date range, vehicle, reason code, status, resolver
- Sort by any column

**Entry Points:**
- "Open reconciliation queue" from 05.1, 05.2, 05.4 footers
- Direct URL `/audit/reconciliation`
- Surfaced inside ops-watch alerts when divergence count exceeds Phase 2 threshold

**Exit Points:**
- Click on a divergence row → [05.6-divergence-detail](../05.6-divergence-detail/05.6-divergence-detail.md)
- "Open audit row" link → [05.1-audit-export-view](../05.1-audit-export-view/05.1-audit-export-view.md)
- Filter / sort interactions stay on this page

## Technical Notes

- **Data shape:** `GET /api/audit/reconciliation?from={date}&to={date}&filters[]={...}` returns paginated reconciliation rows + composite stats. Server pre-computes the composite stats from the filter set so the client doesn't re-aggregate.
- **Auto-reconciler:** background job that, after each paper sign-off is uploaded, compares against the corresponding shadow row and writes a reconciliation row (matched or divergent). Typical latency: <2 minutes. If a paper has no shadow (or vice versa), reason = `missing`.
- **Resolver IdP-binding:** when a human resolves a divergence (annotates it, files a correction, escalates to ops), the resolution event captures their IdP. The `Resolver` column shows the human's display name (or "auto" if the reconciler resolved without human intervention — typically the `net` case, where the depot device synced late but the shadow row eventually matched the paper).
- **Phase 2 threshold check:** server-side. The threshold values come from the pilot config (`unresolved_per_week_threshold: 2`, `time_to_resolution_threshold_hours: 24`). Each reconciliation row carries a flag indicating whether it's above-threshold. The composite stats and the table both reflect these flags.
- **Sorting and pagination:** server-side. Default page size 50; "Show more" loads next page.
- **Print stylesheet for audit-committee report:** the print view is a compact table-only render with composite stats at top — used for monthly Renate-to-her-committee artefact.
