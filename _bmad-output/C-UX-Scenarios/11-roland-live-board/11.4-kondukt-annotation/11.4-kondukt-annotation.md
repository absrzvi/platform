## Overview

**Page Purpose:** When Roland clicks a Kondukt annotation banner on Live Board, the annotation expands inline to show the full claim, the cited IntentPackets and MCP queries, the Validation Receipt details, and a list of affected trains. This is the proof-surface: Roland sees that the Kondukt recommendation passed validation, what evidence it's built on, and which specific trains are involved. The Recommendation Provenance Replay archive triplet is reachable from here (deep-link).

This is the surface that converts a glanceable annotation into an actionable triage decision. Roland either agrees with the pattern (clicks a train to investigate further → 11.5) or dismisses the annotation (which is itself recorded as an operator interaction per FR32).

**User Situation:** Roland has scanned Live Board, noticed the *"Recurring pattern: 3 of 8 active CCTV faults share hardware revision A7. Confidence: HIGH."* banner, and clicked it to expand. He wants to verify the claim before acting on it.

**Success Criteria:**
- Full annotation legible without further expansion
- All cited IntentPackets visible + clickable (deep-link to packet detail in archive)
- Validation Receipt status visible — schema/citation-trace/advisory-language results
- Affected trains listed with severity + position + click-through to Kanban
- Replay archive triplet deep-link present
- Dismiss-with-reason affordance available (records FR32 interaction)

**Entry Points:**
- Click on annotation banner in 11.2 (subsystem-pivot) — banner sits in subsystem column
- Click on annotation banner in 11.3 (train-pivot) — banner floats near affected trains
- Direct deep-link `?annotation_id={recommendation_id}` (rare; for support ticket investigation)

**Exit Points:**
- Click a train in the affected-trains list → [11.5-click-through-to-kanban](../11.5-click-through-to-kanban/11.5-click-through-to-kanban.md)
- Click Replay archive deep-link → opens new tab to archive UI (scenario 05.7 entry point)
- Click "Dismiss" → annotation collapses; dismiss-with-reason recorded; returns to 11.2 or 11.3
- Click outside expanded annotation → collapses without dismissing (state preserved for revisit)
- Esc key → collapses without dismissing

## Components

### Header

| Element | Content | Behaviour |
|---|---|---|
| Annotation title | "KONDUKT ANNOTATION — RECURRING PATTERN DETECTED" (or type-specific) | Static; type derived from Kondukt skill that emitted it |
| Close button (×) | Top-right | Collapses without dismissing (state preserved) |

### Annotation Body

| Element | Content | Behaviour |
|---|---|---|
| Pattern statement | The skill's primary claim verbatim (validated, advisory-only language) | Static |
| Confidence pill | HIGH / MEDIUM / LOW per FR27 | Visual; design-system confidence pill |
| Validation status | Three ticks (✓) or warnings (⚠) for schema / citation-trace / advisory-language linter | Each clickable → hover-popover with detail of that validator pass |
| Source summary | One-line description of what the skill examined | Static |

### Affected Trains List

| Element | Content | Behaviour |
|---|---|---|
| Severity-band swatch | Per train's worst severity | Visual |
| Train ID | "4736-101" etc. | Click → redirect to that train's Kanban detail card (11.5) |
| Route / Location | Short label — current position or "In depot {workshop}" if static | Static |
| Severity badge | HIGH / MEDIUM / LOW | Visual |
| Time-since-fault | Relative ("4h since fault") | Live-updated |
| → arrow | Affordance for click-through | Click = same as Train ID click |

### Cited Evidence

| Element | Content | Behaviour |
|---|---|---|
| IntentPacket count + link | "14 IntentPackets across the last 14 days [view in archive]" | Click "view in archive" → opens new tab to archive UI filtered to these packets |
| Skill reference | "Recurring-failure detection skill v1.0 [view skill spec]" | Click "view skill spec" → opens Kondukt skill registry inspection (scenario 15 surface) — versioned constitution + schema + validator rules |
| Recommendation triplet ID | "Provenance Replay triplet: rcmnd_2026_05_25_06_47_a7c…" + "open in archive" affordance | Click → opens scenario 05.7 archive surface; deep-link includes the triplet ID |

### Action Bar (bottom)

| Element | Content | Behaviour |
|---|---|---|
| Dismiss with reason | Dropdown button | Click → expands reason menu: "Already triaged" / "Not actionable today" / "False pattern" / "Other (with text)". Selection records an `operator_action` row with `action_type='dismissed'` + reason (FR32). Annotation collapses; banner on Live Board removed for current session. |
| Investigate first train (or "Investigate {train_id}") | Primary CTA | Click → equivalent to clicking the first train in the affected-trains list. Redirects to 11.5 → scenario 12 entry. |

## State Variants

| State | Trigger | Visual |
|---|---|---|
| Default expanded | Roland clicked the banner | Full overlay as above, all elements present |
| Validation warning | Validator passed with non-fatal warnings (e.g. low-confidence schema, MEDIUM-confidence citation) | One or more validator badges show ⚠ instead of ✓; clicking the warning badge shows the validator detail |
| Validation failed (annotation should not have reached this page) | Edge case — if surface accidentally renders a suppressed annotation | Banner: "This annotation was suppressed by the validator and should not be visible. Reason: {reason}. Please contact Nomad ops." Affected trains list still shown (operator may still want to triage; harness records this as a malformed-display incident per FR57 security event) |
| BYOK provider degraded | Roland clicked annotation just as provider degraded | Existing annotation content readable; Kondukt-related dynamic content (e.g. "view skill spec") may show pending state |
| Affected trains list overflow | Skill identified >10 trains | Shows first 10 + "+ N more (scroll)" affordance |
| Single affected train | Skill pattern applies to 1 train only | Affected trains list shows 1 row; "Investigate first train" CTA becomes "Investigate {train_id}" |

## Behaviour Notes

- **Inline overlay positioning:** anchored to the source banner; if the overlay would extend beyond viewport, it flips to anchor from a different edge. Implementation: floating-ui or popper.js style positioning.
- **Background dimming:** subtle (10–15% opacity); does NOT modal-block — Roland can still see the map context and is reminded that the annotation is one piece of information about the broader fleet state.
- **Replay archive deep-link** opens in a new tab so Roland doesn't lose the Live Board context.
- **Dismiss-with-reason recording** — operator action recorded per FR32 with reason text; affects future Kondukt suggestion behaviour over time (the suppression / acceptance signal feeds the calibration loop).
- **Keyboard:** Enter on a train row redirects; Esc closes overlay; Tab cycles between elements; D opens dismiss-with-reason dropdown.

## Open Questions

- **Dismiss-with-reason taxonomy** — the 4 options listed are working assumption. Phase 4 should validate against Roland's pilot use. May need 5th option "Wrong subsystem" or "Wrong train" for noisy-pattern handling.
- **Visual dimming intensity** for the background — 10% vs 15% vs 20%. Trade-off: too dim loses the "you're still on Live Board" framing; too light reduces overlay readability. Phase 4 confirms.
- **Multi-skill annotation aggregation** — what if two different Kondukt skills produce annotations for the same subsystem cluster (e.g. recurring-failure + Daily Brief). Working assumption: each surfaces independently. Future state may aggregate.
- **"Suppressed but visible" edge case** — should this surface ever appear in MVP? Working assumption: no, it's a defence-in-depth state. If it appears, FR57 security event fires.
- **Replay archive open-in-new-tab vs in-place** — currently spec'd as new tab to preserve Live Board context. Alternative: side-panel split-view. New tab is simpler for MVP.
