## Overview

**Page Purpose:** Live Board reoriented around subsystems. The right edge of the map carries an aggregated severity column per subsystem (CCTV / PIS / intercoms / AFZ / audio / Wi-Fi / switches / Frontkupplung / energy meter / Service FIS/CCTV / Zugfunk / TCMS / Stadler diagnostic / Firewall RCU). Roland sees per-subsystem severity at a glance — "how is CCTV across the whole fleet today?" — rather than per-train. Map markers retain severity bands but the subsystem aggregate is the primary insight.

This is the default pivot for first-run users (showcases the multi-subsystem visibility USP). For returning users, this view appears only if they last used it (per FR3 per-user pivot state restore).

**User Situation:** Roland chose subsystem-pivot intentionally (or it's first-run). He wants the macro view of *which subsystems are degraded today* before drilling into individual trains.

**Success Criteria:**
- Per-subsystem severity aggregate legible without scrolling within the column
- Click on subsystem column filters map markers to trains with active fault in that subsystem (in <300ms perceived)
- Kondukt annotations cluster appropriately near the subsystem they relate to
- All 14 subsystems visible in the column (without scrolling on ≥1080 vertical viewports); compact at lower viewports

**Entry Points:**
- From 11.1 default landing if subsystem-pivot is the current state
- From 11.3 train-pivot via pivot toggle in Fleet Overview Bar
- Direct deep-link `?view=live-board&pivot=subsystem` (URL state)

**Exit Points:**
- Pivot toggle → [11.3-train-pivot](../11.3-train-pivot/11.3-train-pivot.md)

## Components

### Subsystem Column

**Visible by default** (subsystem-pivot view; hidden in train-pivot).

Top of column: heading "Subsystems" + small sort/filter affordance.

Each row in the column:

| Field | Content | Behaviour |
|---|---|---|
| Severity-band swatch | Coloured strip on the left — most severe active fault for that subsystem across the fleet | Visual scan; no interaction |
| Subsystem name | Operator-friendly (CCTV, PIS, intercoms, AFZ, audio, Wi-Fi, switches, etc.) — **no VLAN IDs** per FR2 | Click → filter map markers to trains with active fault in this subsystem (state visible in Fleet Overview Bar as filter pill) |
| Active fault count | Integer (e.g. "8 active") | Live-updated |
| Severity distribution pips | Up to 5 coloured dots showing distribution of severity levels (mostly amber + few red, etc.) | Visual; no interaction |
| Kondukt annotation marker | Small icon if Kondukt has surfaced a pattern relevant to this subsystem | Click → expands to 11.4 annotation detail |

**Row order:**
- Primary: severity-band colour (worst first → CRITICAL → HIGH → MEDIUM → LOW)
- Secondary: active fault count (descending)
- Tertiary: alphabetical

Tie-breaker example: if CCTV and PIS both have HIGH severity, the one with more active faults is on top.

**Subsystem taxonomy (MVP, from IP allocation reference — operator-friendly names only):**

| Display name | Internal subsystem class | Notes |
|---|---|---|
| CCTV | `cctv_camera`, `rear_view_camera` (combined under "CCTV") | Includes Überw. Kamera + Rücksehkamera |
| PIS | `interior_display` (Bildschirm), `side_display` (Seitenanzeige), `front_display` (Frontanzeige) | Three sub-types aggregated as "PIS" |
| intercoms | `intercom` (Sprechstelle), `intercom_prm` (Sprechstelle PRM) | Standard + PRM variant aggregated |
| AFZ | `passenger_counter` | Aut. Fahrgastzähler |
| audio | `audio_amp`, `audio_distribution_unit` (ADU) | Amplifiers + ADU |
| Wi-Fi | `wifi_access_point` | Access points across multi-VLAN trunk |
| switches | `fis_switch` | FIS backbone switches |
| Frontkupplung | `front_coupler_data` | Front coupling data network |
| energy meter | `energy_meter` (Energiezähler) | Single per-train |
| Service FIS/CCTV | `service_fis_cctv` (diagnostic plane) | VLAN 2 — but only operator-friendly name visible |
| Zugfunk | `cab_radio` (ZFR) | Cab radio interface |
| TCMS | `tcms` (received via brain VLAN 7 SNMP forward) | Brain-mediated |
| Stadler diagnostic | `stadler_rdc`, `stadler_obs`, `firewall_rcu` | Three components grouped; Stadler's plane |

Total 13 displayed; ordered per the rule above.

### Map Area (subsystem-pivot variant)

Map markers behave per [11.1](../11.1-default-landing/11.1-default-landing.md) with these refinements:
- When a subsystem column row is selected/filtered, markers for trains *without* an active fault in that subsystem dim to 30% opacity. Selected trains remain at 100%.
- Marker tooltip on hover shows the top-fault in the *currently-filtered* subsystem if a filter is active, else the top-fault overall.

### Kondukt Annotation Banner (subsystem-pivot positioning)

- Annotations float **near the affected subsystem column row**, not over the map. This is the subsystem-pivot-specific positioning.
- Compact pill: 1-line summary + Validation Receipt badge.
- Click → expands inline → [11.4](../11.4-kondukt-annotation/11.4-kondukt-annotation.md).

## State Variants

| State | Trigger | Visual |
|---|---|---|
| Default | Subsystem-pivot active; no subsystem filter | All trains visible at 100%; subsystem column shows all 13 rows |
| Subsystem filter active | Roland clicked CCTV row | Map markers filtered to trains with active CCTV fault; non-matching trains at 30% opacity; CCTV row visually highlighted in column; filter pill "CCTV faults only" added to Fleet Overview Bar |
| Multiple subsystem filters | Roland Cmd+click-ed CCTV + PIS rows | Union semantics: trains with active CCTV OR PIS fault visible at 100%; two filter pills |
| Kondukt annotation in column | Kondukt has surfaced a pattern related to this subsystem | Icon visible on the subsystem row; click expands to 11.4 |
| All subsystems healthy | Zero active faults across all subsystems | Column shows all rows with grey severity bands + "0 active"; map shows trains with neutral severity bands |
| Kondukt suppressed/degraded | FR20 — BYOK LLM unreachable | No annotation markers on the column; FR20 banner visible per 11.1 |

## Behaviour Notes

- **Filter persistence:** subsystem filters persist within the session but NOT across sessions (FR3 persists pivot state, not filter state).
- **Filter pills in Fleet Overview Bar:** when one or more subsystems are filtered, pills appear with × to clear. Clicking pill × removes that subsystem from filter; clicking "Clear all filters" removes all.
- **Keyboard:** arrow up/down on subsystem column navigates rows; Enter toggles filter; Esc clears active filter.
- **Performance budget:** filter toggle re-render <300ms.

## Open Questions

- **Compact vs expanded subsystem column** at narrower viewports (1280–1440 width) — should the column collapse to a thin icon-only strip with hover-expand, or always show full row content? Working assumption: full row content; column shrinks proportionally. Phase 4 confirms.
- **Severity-distribution pips quantification** — show actual numeric distribution on hover (e.g. "CRITICAL: 1, HIGH: 4, MEDIUM: 3")? Or just the dots? Working assumption: dots + hover-tooltip with numbers.
- **Subsystem icon set** — should each subsystem have a small icon (camera for CCTV, screen for PIS, speaker for audio, etc.)? Working assumption: text-only labels for MVP, icons as a Phase 4 polish item if it improves scan speed.
- **Filtering to zero results** — if Roland filters by a subsystem with no active faults, what does the empty state look like? Working assumption: map shows all trains dimmed to 30%; banner above status strip: "No trains have active faults in selected subsystems."
