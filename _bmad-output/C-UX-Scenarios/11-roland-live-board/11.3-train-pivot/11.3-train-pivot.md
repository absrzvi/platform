## Overview

**Page Purpose:** Live Board reoriented around individual trains. The subsystem column collapses; each train marker on the map carries an inline subsystem-state visualisation (a small ring of dots or stacked indicator) showing per-train per-subsystem severity. Roland sees *which trains have the most subsystem-fault load* — useful when investigating a specific train cluster or when the question is "what's wrong with that train?"

The right edge of the map carries a **trains-with-active-faults panel** instead of subsystems — a sortable list of trains with their fault counts and dominant severity. Roland can scan the list, see the cluster the Kondukt annotation referenced, and click into a specific train.

**User Situation:** Roland has noticed the Kondukt annotation about three CCTV faults sharing hardware revision A7 (from 11.2 subsystem-pivot view). He pivots to train-pivot to see *which trains* are affected and where they are on the map.

**Success Criteria:**
- Per-train subsystem state legible without expansion (small inline visualisation per marker)
- Trains-with-active-faults list sortable by: fault count, dominant severity, time-since-fault, geography
- The cluster of three CCTV-A7 trains is visually distinguishable (same severity band + nearby positions if route-correlated)
- Click on train marker OR list row redirects to that train's Kanban detail card

## Components

### Trains-With-Faults Panel

Visible by default in train-pivot view; hidden in subsystem-pivot.

Top of panel: heading "Trains with active faults" + sort affordance.

Each row in the panel:

| Field | Content | Behaviour |
|---|---|---|
| Severity-band swatch | Coloured strip — train's worst-severity active fault | Visual |
| Train ID | E.g. "4736-101" | Click → redirect to Kanban detail card (11.5) |
| Fault count | E.g. "3 faults" | Live-updated |
| Dominant severity badge | HIGH / MEDIUM / LOW | Visual |
| Subsystem chips | Up to 4 small text chips of affected subsystems (CCTV, PIS, audio, …); overflow as "+N" | Click subsystem chip → filter list to trains with that subsystem affected (intersection with current list) |

**Row order (default):**
- Primary: severity (worst first)
- Secondary: fault count (descending)
- Tertiary: time-since-most-recent-fault (recent first)

Roland can change sort via the sort dropdown: severity / fault count / time-since-fault / route position.

### Map Area (train-pivot variant)

Map markers behave differently than in 11.2:
- **Severity-band ring** around each marker (not just the marker fill) — colour = train's worst-severity active fault.
- **Inline subsystem-state visualisation** inside each marker — small stacked-dot pattern or radial pie showing per-subsystem severity:
  - Up to 9 subsystems indicated as small dots/pips arranged in a 3×3 grid inside the marker (or radial)
  - Each pip colour = severity of that subsystem (colour map per design system)
  - Empty/grey = no active fault in that subsystem
  - Markers without any active faults: just the marker circle, no pips, neutral colour

This is denser per-marker than the 11.2 view but is what train-pivot is for — *per-train* state.

- **Cluster behaviour:** at low zoom, markers cluster (Mapbox-style); cluster bubble shows aggregate fault count + worst severity for the cluster.

### Kondukt Annotation Banner (train-pivot positioning)

- Annotations float **near the affected train markers**, not over a column. This is the train-pivot-specific positioning.
- If multiple trains share an annotation (the recurring-CCTV-A7 case), the annotation banner sits between the affected markers with a visual connector (subtle line linking the trains).
- Compact pill: 1-line summary + Validation Receipt badge.
- Click → expands inline → [11.4](../11.4-kondukt-annotation/11.4-kondukt-annotation.md).

## State Variants

| State | Trigger | Visual |
|---|---|---|
| Default | Train-pivot active; no row filter | All trains visible; trains-with-faults panel shows all trains with ≥1 active fault, sorted by severity |
| Subsystem chip filter | Roland clicked a subsystem chip on a row | Panel + map filtered to trains with that subsystem affected; filter pill in Fleet Overview Bar |
| Multi-subsystem-chip filter | Roland Cmd+click-ed multiple chips | Intersection semantics: trains affected by all selected subsystems |
| Map cluster click (low zoom) | User clicked a marker cluster bubble | Zoom in one level; cluster expands; no redirect (only individual markers redirect) |
| Train marker hover | Mouse over individual train marker | Tooltip: train ID + position + fault count + dominant subsystems |
| Train marker click | User clicked individual marker | Redirect to 11.5 → scenario 12 (Kanban detail card) |
| Kondukt annotation cluster | Kondukt pattern affects multiple trains | Visual connector lines between affected markers; floating banner sits between them |
| Kondukt suppressed/degraded | FR20 — BYOK LLM unreachable | No banners; previous annotations remain visible in archive (per 11.1 state) |

## Behaviour Notes

- **Filter semantics** in train-pivot (intersection) differ from subsystem-pivot (union). Rationale: in subsystem-pivot, filters answer "which trains have any of these subsystems failing?" (union — wider net). In train-pivot, filters answer "which trains have ALL of these subsystems failing?" (intersection — narrower). The intersection makes the cluster Kondukt detected (3 trains, same subsystem) easy to surface.
- **Marker pip density** — 9 pips per marker max in MVP (the 9 most-active subsystems on that train). Trains with >9 affected subsystems show an "overflow" indicator (rare in practice).
- **Subsystem chip ordering** in the trains-list row: most-severe subsystem first.
- **Keyboard:** arrow up/down on trains-list navigates rows; Enter selects (redirect to Kanban); arrow keys on map pan; `f` zooms to all trains with active faults.

## Open Questions

- **Marker pip layout** — 3×3 grid vs radial vs vertical stack? Visual density vs scanability trade-off. Working assumption: 3×3 grid (most compact + recognisable). Phase 4 confirms with mockups.
- **Cluster bubble information density** — show aggregate fault count + worst severity at minimum. Show subsystem aggregate too? Working assumption: count + severity only; cluster click zooms in for detail.
- **Trains-with-faults panel max rows visible** — viewport-dependent. At ≥1080 vertical, 12–18 rows visible without scroll. Scroll affordance below.
- **"Trains without active faults" filter toggle** — does Roland ever want to see healthy trains too? Working assumption: panel only shows trains with ≥1 active fault by default; toggle to "include healthy" available but off by default.
- **Multi-train Kondukt annotation visual connector** — line, dotted line, subtle curve? Should not dominate the map. Phase 4 confirms aesthetic.
