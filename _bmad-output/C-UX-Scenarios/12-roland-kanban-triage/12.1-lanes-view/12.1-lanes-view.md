## Overview

**Page Purpose:** Roland's continuous all-day surface for triaging the fleet's open faults by *neglect-risk* — the decay-without-attention discipline. Four horizontal lanes: New / Investigating / Recommendation Pending / Cleared. Cards within each lane sort by Kondukt-computed neglect-risk score (highest first). Each card carries a Kondukt-proposed lane move with provenance + Validation Receipt — Roland accepts or modifies; operator action logged.

This is also the **depth surface** entry point — clicking a card title expands it to the train detail (primary disclosure per FR10); clicking a component row reveals secondary disclosure (sub-page 12.4).

**User Situation:** Default Kanban view. Roland could be arriving from any of: Live Board click-through (12.5 redirect — common case), top-nav direct from another harness view, browser deep-link from a colleague or from his own session resume. View state (lane filters, sort preference, expanded cards) restored per-user.

**Success Criteria:**
- All four lanes visible without horizontal scroll on ≥1920 viewport
- Cards in each lane sorted by neglect-risk (highest first); manual sort toggle available
- Kondukt-proposed lane-move banner visible on every card with provenance + Validation Receipt
- Filter pills active in fleet overview bar; filter state persisted per-user
- Cards selectable / expandable via click; drag-and-drop functional for lane moves
- Real-time updates: new cards appear in New lane via SSE; moved cards animate to destination lane

**Entry Points:**
- From scenario 11.5 — Live Board click-through (URL: `/kanban?train={id}&detail=expanded&from=live-board`)
- Direct top-nav from Live Board or Kondukt
- Browser back from a different view
- Direct deep-link URL `/kanban` or `/kanban?filter=cctv`

**Exit Points:**
- Click "Accept move" on a card → [12.2-lane-move-acceptance](../12.2-lane-move-acceptance/12.2-lane-move-acceptance.md)
- Click card title or expand chevron → [12.3-train-detail-primary](../12.3-train-detail-primary/12.3-train-detail-primary.md)
- Click "Modify" on LOW-confidence proposal → [12.5-low-confidence-on-card](../12.5-low-confidence-on-card/12.5-low-confidence-on-card.md)
- View nav → Live Board or Kondukt
- Click filter pill → re-renders with filter applied
- Drag card to different lane → triggers manual lane move; logged as `operator_action`

### Kanban Card (template — appears N times per lane)

Each card carries the following structure:

```
┌─────────────────────────────────┐
│ ▒ 4736-101  CCTV                │  ← severity-band swatch + train ID + dominant subsystem
│ Wien Hbf → Salzburg              │  ← route / location
│ ───────────────────────────────  │
│ neglect-risk: HIGH               │  ← Kondukt-computed neglect-risk badge
│ time-since-fault: 4h 23m         │  ← oldest active fault age
│ ───────────────────────────────  │
│ Kondukt proposes:                │  ← Kondukt's lane-move proposal banner
│  ↗ Move to Investigating         │
│  ✓ schema · ✓ citation · ✓ adv. │  ← Validation Receipt (3-check)
│ [Accept]  [Modify]  [Reject ▼]  │  ← Action buttons (Accept primary)
│ ───────────────────────────────  │
│ Subsystems affected: CCTV (1)    │  ← subsystem chips
│ ↓ Expand for component detail    │  ← expand affordance → 12.3
└─────────────────────────────────┘
```

**Visual hierarchy:**
- **Severity-band swatch (left edge):** 6px coloured strip — train's worst-severity active fault. CRITICAL red / HIGH orange / MEDIUM amber / LOW yellow.
- **Train ID + dominant subsystem:** card header. Bold weight. Train ID is the click-target for card expansion (12.3).
- **Route / location:** smaller text below header. For depot trains: "In depot {workshop}".
- **Neglect-risk badge:** HIGH / MEDIUM / LOW pill. Kondukt-computed.
- **Time-since-fault:** oldest active fault age, relative format.
- **Kondukt proposal banner:** the lane-move proposal. Validation Receipt 3-check icons.
- **Action buttons:** Accept (primary; visible affordance) + Modify (secondary) + Reject with dropdown reason.
- **Subsystem chips:** affected subsystems on this train. Click chip to filter Kanban to other trains with that subsystem affected.
- **Expand affordance:** small chevron at card bottom; click to expand to 12.3.

### Card Variants by Lane

| Lane | Card variants |
|---|---|
| New | Kondukt's proposal: "Move to Investigating" (typical). LOW-confidence proposals route through 12.5 variant. |
| Investigating | Kondukt's proposal: "Move to Recommendation Pending" — typically when investigation produces a concrete proposed action (depot scheduling, parts order, vendor escalation) |
| Recommendation Pending | Kondukt's proposal: "Move to Cleared" — when downstream-state polling (FR33) verifies the action was taken and outcome is resolved |
| Cleared | No Kondukt proposals; cards are read-only summary; ages out after N days (working assumption: 14 days, then archived) |

### LOW-Confidence Variant on Card

When Kondukt's proposal confidence is LOW:
- Confidence pill rendered as outline-only (per design system LOW-DISABLE rule)
- **"Accept" button is non-interactive** (`aria-disabled="true"` + tooltip: "Low-confidence recommendations require operator review — use Modify.")
- **"Modify" button is the only execution path** — cursor pre-placed in justification field on click
- modification_distance captured on commit
- Detailed in sub-page 12.5

### Drag-and-Drop Lane Move

| Element | Behaviour |
|---|---|
| Drag handle | Entire card body draggable (cursor: grab on hover) |
| Drag visual | Card shadow + slight rotation; original position shows placeholder outline |
| Drop zones | Each lane is a valid drop target; visual highlight when dragged card hovers over a target lane |
| Drop commit | On drop → card relocates to target lane; operator action recorded with `action_type='kanban_lane_move_manual'`; if Kondukt had proposed a different move, the manual override is flagged in audit (Roland's modification rate signal) |
| Keyboard alternative | Tab focus card → arrow keys move card between lanes (left/right); Enter commits |
| Cancel during drag | Esc cancels drag; card returns to original position |

| State | Trigger | Visual |
|---|---|---|
| Default | Lanes loaded, no filter active | All 18 cards distributed; sort by neglect-risk; Kondukt proposals visible |
| Filter active | Roland applied subsystem / lane / neglect-risk filter | Cards filtered; filter pill visible in Fleet Overview Bar; lane counts updated |
| Card auto-expanded (from Live Board click-through) | Entry from scenario 11.5 with URL param `?train={id}&detail=expanded` | Specified card auto-expanded at primary disclosure (12.3 state); page scroll to centre |

| Real-time card move (by another user) | Another operator (rare in MVP, but possible) moved a card | Card animates to new lane; subtle indicator: "moved by {operator} just now" |

## Behaviour Notes

- **Live updates** via SSE. Card additions, moves, expansions all stream. Throttled to max 1 update per card per 2 seconds.
- **Card expansion state** persists for the session in client localStorage. Expanding card A then navigating to Live Board and back preserves A's expansion. Cross-session: not persisted (working assumption; Phase 4 confirms).
- **Filter persistence** — `filter` state persisted per-user server-side (`user_preference` table). Restored across devices.
- **Sort persistence** — same as filter.
- **Drag-and-drop accessibility** — keyboard alternative is mandatory (FR9). Arrow keys move card; Enter commits; Esc cancels. Screen reader announces lane changes.
- **Performance budget:** initial render <2s; card expansion <100ms; drag-and-drop animation <16ms per frame (60fps).
- **Per-card Kondukt proposal data fetched lazily** — when a card is in viewport (using IntersectionObserver), its proposal banner data loads; off-viewport cards have proposal data stubbed (saves render cost).

## Open Questions

- **Lane width default + per-user customisation** — equal-width default; Roland may want Cleared collapsed by default. Working assumption: equal-width; per-user collapse / resize preferences persisted.
- **Card auto-expansion on Live Board entry — animation** — should the expansion be instant or animated (e.g. 200ms expand)? Working assumption: instant for the auto-expansion to feel responsive; manual expansion (chevron click) gets a 150ms ease.
- **Manual sort override semantics** — when Roland switches sort to e.g. "train ID alphabetical", do filters still apply? Working assumption: yes; sort and filter are orthogonal.
- **Recommendation Pending lane signal type** — what specifically marks a card as "Recommendation Pending"? Working assumption: Kondukt has emitted a structured recommendation that requires operator action outside the harness (depot scheduling, parts order, vendor escalation, HAFAS update) which hasn't been logged as taken yet. Phase 4 confirms with `lane_signal_type` enum.
- **Cleared lane aging** — when does a Cleared card disappear? Working assumption: 14 days, then archived to a separate "Cleared archive" view. Phase 4 confirms.
- **Drag-and-drop accessibility ARIA labels** — every lane and card needs proper ARIA role + label. Working assumption: list semantics for lanes, listitem for cards. Phase 4 confirms.
- **Card visual treatment when expanded inline within a lane** — does the expanded card expand the lane's width or only the card's height? Working assumption: height only; lane width fixed.
