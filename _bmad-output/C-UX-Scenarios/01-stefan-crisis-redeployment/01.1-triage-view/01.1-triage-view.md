## Overview

**Page Purpose:** Stefan's default monitoring state — a 70/30 split (triage feed dominant, secondary contextual panel) showing all live fleet disruption signals with cost-impact one-liners visible on every card before he expands anything. The single page from which crisis, query, search, and RAO-generation flows all enter.

**User Situation:** Stefan is mid-shift in the Wien operations centre. He scans this view continuously. The page is the "home base" he returns to after every action.

**Success Criteria:**
- Cost-impact one-liner is visible on every triage card without click
- New incidents appear in the feed within 3 seconds of upstream IntentPacket ingest (FR per PRD: < 3s initial render under PoC fleet size 10–20 vehicles)
- Persistent chrome includes Stefan's IdP identity + shift duration counter

**Entry Points:**
- Default page after IdP-authenticated login
- Return target after every other Fleet Manager AI Interface page (modal close, RAO mark-executed, search exit, handover-complete)
- Direct URL: `/triage` (TanStack Router route)

**Exit Points:**
- Disruption fires → [01.2-crisis-modal](../01.2-crisis-modal/01.2-crisis-modal.md)
- Stefan clicks a card → card expand surface (scenario 02 path; not in scenario 01)
- Stefan clicks "Generate handover" → scenario 09.1
- Kondukt becomes unavailable → degraded variant (scenario 10.1)

### Section: Triage Feed (70% width)

**OBJECT ID:** `triage-feed`

| Property | Value |
|----------|-------|
| Purpose | Live disruption feed, virtualised scrollable list of triage cards |
| Component | Virtualised list (`@tanstack/react-virtual`) for ≥50 simultaneous cards |
| Padding | `space-lg` (24px) all sides within the 70% pane |
| Element gap | `space-md` (16px) between cards |

**Sort order (LOCKED 2026-05-25 via party mode — Sally + Mary converged):** **Severity-banded, newest-first within band.** Bands: HIGH → MEDIUM → LOW. Within each band, chronological newest-first. LOW band capped at 5 visible items with "Show more (N)" affordance — otherwise becomes noise on hour 11 of a shift. (Crisis modal already pre-empts CRITICAL+`request_landside_tasking`, so feed handles non-crisis.) Evidence basis: control-room cognitive-load research (Vicente, Endsley situation-awareness Level 2) — operators degrade on scanning under long shifts but stay sharp on recognition; banded hybrid preserves both severity-as-attention-budget and recency-as-pattern-detector.

| State | When | Appearance | Actions |
|-------|------|------------|---------|

## Technical Notes

- **Performance contract:** Initial render under PoC fleet size (10–20 vehicles) < 3s (PRD FR).
- **Streaming:** Cards arrive via SSE from `/api/intentpackets/stream` (downstream of MQTT). Idempotent insertion on `packet_id`.
- **Virtualisation:** Use `@tanstack/react-virtual` to handle ≥50 cards without scroll jank.
- **Cost-impact computation:** Pre-computed server-side from IntentPacket + HAFAS state at the moment of triage card emission. Not a client-side LLM call. Server response includes `cost_impact_eur` + `cost_impact_confidence` + `cost_impact_explanation_short`.
- **Severity band rendering:** CSS-only (no SVG). Width `space-xs` (8px), full card height.
- **Card click handler:** Routes to scenario 02 path (card expand) by default. In scenario 01 path, the *modal* fires on disruption, not card click — scenario 01 entry is push-driven by upstream event.
- **Kondukt status polling:** `/api/kondukt/health` every 30s. Failure threshold for degraded banner activation: 5 min sustained (Phase 4 open question — confirm with Phase 3 scenario 10).
