## Overview

**Page Purpose:** The validated Daily Brief rendered in the Kondukt canvas. The harness has just completed: LLM draft from the customer's BYOK provider → schema validator passed → per-claim citation-trace validator passed → advisory-language linter passed → emission committed. The brief Roland reads has provenance attached to every claim; each finding cites the IntentPackets the LLM used to support it. The recommendation triplet (Envelope + Provenance Bundle + Validation Receipt) is now in the archive.

This is the most important surface in the harness for the constitution-as-contract claim. Renate's outside auditor will see this rendered version + run the Replay CLI to verify it deterministically against the stored triplet. Roland sees the validator passes inline — three green checks (schema / citation-trace / advisory-language) per finding.

**User Situation:** Roland has just invoked the Daily Brief skill (13.1). The skill completed in ~15 seconds. The canvas now shows the brief — three findings, one HIGH confidence + one HIGH + one MEDIUM. He starts reading.

**Success Criteria:**
- Brief structure clear: recommended-focus section at top → findings list → "awaiting your input" section at bottom (if applicable)
- Each finding shows: severity badge + confidence pill + claim text + citation chips + Validation Receipt status (3-check)
- All claims are advisory-only language (no imperative verbs)
- Click on a citation chip opens the citation's hover-popover (preview); click "view provenance" → Replay archive
- Brief is text-selectable for copy/paste (operators may want to share findings with colleagues)
- "Acknowledge — go to Kanban" affordance at bottom; navigates with the most relevant filter pre-applied
- Recommendation triplet ID visible (subtle, in metadata footer) — gives Roland a way to reference this brief later

**Entry Points:**
- From 13.1 after validator passes
- Browser deep-link from previous session (rare — `/kondukt?session=…` or by recommendation_id)
- Replay archive deep-link from scenario 05.7 (when Renate's auditor is inspecting this specific brief)

**Exit Points:**
- Hover on citation chip → tooltip preview (no navigation)
- Click citation chip → [13.3-confidence-pill-provenance](../13.3-confidence-pill-provenance/13.3-confidence-pill-provenance.md) (provenance inspection)
- Click "View provenance" link on a finding → opens Replay archive triplet detail in new tab (cross-link to scenario 05.7)
- Click "Acknowledge — go to Kanban" → navigates to scenario 12 with filter pre-applied; operator action recorded
- Click "Acknowledge — go to Live Board" → navigates to scenario 11 with relevant filter
- Click "Re-run Daily Brief" → invokes skill again (back to 13.1 running state)
- Click another skill button → that skill's flow
- Navigate to Live Board / Kanban without acknowledging → exits without action record

## Components

### Brief Metadata Header

| Element | Content | Source |
|---|---|---|
| Skill name + timestamp | "Daily Brief · 06:30 · 2026-05-25" | Emission timestamp + skill name from triplet Envelope |
| Confidence aggregate badge | "⊕ HIGH+MED" — summarises confidence distribution across findings | Computed from per-finding confidence pills |
| Skill version | "v1.0" — links to skill changelog (Phase 1.5) | Envelope field |
| Recommendation triplet ID | "rcmnd_2026_05_25_06_47_a7c…" (truncated; clickable to expand to full ID) | FR29 Envelope |
| Validation Receipt status (3-check) | "✓ schema · ✓ citation-trace · ✓ advisory" — aggregated brief-level | FR23–FR26 |
| Click on triplet ID | Opens Replay archive in new tab with this triplet preloaded | Cross-link to scenario 05.7 surface |

### Recommended Focus Section

| Element | Content |
|---|---|
| Section header | "📍 RECOMMENDED FOCUS TODAY" |
| Focus text | Synthesis from the highest-severity / highest-neglect-risk findings — 1–3 sentences |
| Visual emphasis | Larger font + accent colour; first thing Roland sees |
| No citations here | Citations live in the per-finding blocks below; focus section is the synthesis |

### Findings Section

Section header: "◆ FINDINGS (N)" — N is the count.

Each finding block:

| Element | Content |
|---|---|
| Severity badge | HIGH / MEDIUM / LOW / CRITICAL (per design system severity colours) |
| Confidence pill | HIGH (filled green) / MEDIUM (half-filled amber) / LOW (outline-only slate per LOW-DISABLE rule) |
| Claim text | The LLM's draft, validated through advisory-language linter (no imperative verbs); operator-friendly subsystem names; concrete train IDs + component references |
| Citation chips | Inline chips with IntentPacket IDs in compact format `[pkt_05-25_02-14_DN-014]` — clickable to hover-popover (13.3); each chip is a span with `data-packet-id` attribute |
| Validation Receipt (per-finding 3-check) | `✓ schema · ✓ citation-trace · ✓ advisory-language` — each click expandable to validator-rule detail |
| "View provenance" link | Click → opens Replay archive triplet detail in new tab (this specific finding's evidence) |

**Finding sort order:**
- Primary: severity (worst first)
- Secondary: confidence (HIGH > MEDIUM > LOW)
- Tertiary: subsystem alphabetical

### Awaiting Your Input Section

| Element | Content |
|---|---|
| Section header | "📌 AWAITING YOUR INPUT (N)" |
| List items | In-flight items from previous shift requiring Roland's attention — vendor reply pending, scheduled meetings, audit deadlines |
| Source | Pulled by skill from `operator_action` history + Kanban "Recommendation Pending" lane state |
| Action affordance per item | Subtle "Open" link → navigates to relevant view (Kanban card, Kondukt session, scheduled item) |

This section may be empty for some shifts (e.g. quiet weekend); section hidden entirely if empty.

### Action Affordances (bottom of brief)

| Element | Content | Behaviour |
|---|---|---|
| Acknowledge — go to Kanban (filter pre-applied) | Primary CTA | Navigates to scenario 12 with filter pre-applied based on highest-priority finding (e.g. "subsystem: CCTV + train: DOSTO Neu"). Records operator action `kondukt_skill_ack_and_navigate` per FR32. |
| Re-run Daily Brief | Secondary | Re-invokes the skill (back to 13.1 running state). New triplet emitted. |
| Download brief as .docx | Tertiary | Generates a `.docx` from the brief content + triplet metadata; saves locally; recorded per FR18 (operator artifact download). |

| Component | Content |

## Behaviour Notes

- **Validator outcomes are deterministic.** The same LLM draft + same Provenance Bundle re-validated by the Replay CLI six months later produces the same Validation Receipt. This is the architectural guarantee — Replay archive integrity (FR63 criterion 3).
- **Citation chip data is content-addressed.** Each `pkt_*` ID is a stable identifier; the chip's `data-packet-id` attribute matches the Provenance Bundle's `intent_packets_consumed` list. Auditor can verify any chip's claim against the bundle.
- **Operator interaction logged per FR32:**
  - `kondukt_skill_emission_rendered` (when brief renders for the first time)
  - `kondukt_skill_citation_hovered` (when Roland hovers a citation chip — telemetry signal that the citation is being read, not skipped)
  - `kondukt_skill_provenance_clicked` (when Roland clicks "View provenance" — Replay archive cross-link engagement)
  - `kondukt_skill_ack_and_navigate` (when Roland clicks Acknowledge — primary commit action)
  - `kondukt_skill_artifact_downloaded` (when Roland downloads .docx — artifact production tracking)
- **Text selection allowed** — brief text is selectable; copy/paste to share with colleagues. Telemetry: `kondukt_skill_text_copied` (best-effort detection via clipboard API).
- **Re-run produces a new triplet** — same skill, new invocation, new `recommendation_id`. The previous triplet remains in archive (immutable). The new brief replaces canvas content; previous brief retrievable via "session history" (Phase 1.5; MVP working assumption: canvas single-artifact at a time).
- **Performance:** brief render after validator pass <500ms; citation chip hover popover <200ms.

## Open Questions

- **Findings-per-brief upper bound** — visual treatment when brief has >15 findings? Working assumption: render all but with collapsible "low priority" section; pagination not in MVP. Phase 4 confirms.
- **Confidence aggregate badge in header** — current working assumption: shows worst confidence in brief ("⊕ HIGH+MED" if any MED, "⊕ HIGH+LOW" if any LOW, etc.). Alternative: show distribution percentages. Phase 4 confirms with Roland-test.
- **Citation chip visual format** — `[pkt_05-25_02-14_DN-014]` working assumption. Operators may want shorter (`[02:14 DN-014]`) or more meaningful (`[CCTV-A7 04:23]`). Phase 4 confirms with usability testing.
- **Acknowledge CTA filter pre-application logic** — currently: highest-priority finding's subsystem + train fleet. May need refinement (e.g. union of all HIGH-confidence findings vs single most-severe). Phase 4 confirms.
- **Re-run vs new skill** — when Roland re-runs Daily Brief, the previous canvas content is replaced. Some operators may want comparison view ("diff from previous brief"). Phase 1.5 candidate.
- **Docx export content** — full brief text + triplet metadata in working assumption. Should it include the Provenance Bundle (heavy)? Or just the surface text + triplet ID for traceability? Phase 4 confirms.
- **Already-read indicator persistence** — across sessions? Working assumption: yes, per-user. Phase 4 confirms.
- **Validation Receipt rule-detail popover content** — clicking a green check shows what? Working assumption: the specific validator rule that passed + the data it checked. Phase 4 confirms.
