## Overview

**Page Purpose:** The provenance-inspection surface for a Daily Brief claim. When Roland clicks a citation chip (e.g. `[pkt_05-25_02-14_DN-014]`) in a finding, an inline panel reveals below the finding showing: the cited IntentPacket's full identifier + arrival timestamp + payload structure + brief excerpt + cross-link to the Replay archive triplet that this brief is part of. This is the FR5 surface from the Kondukt context — every claim is reachable through provenance to its source data.

The confidence pill on the finding is the visual surface of confidence; this sub-page covers what happens when Roland verifies the *source* of the claim that the confidence is about.

**User Situation:** Roland is reading the Daily Brief (13.2). A specific claim — "Three DOSTO Neu units exhibited reservation-screen blackouts overnight" — has three citation chips. He clicks the first one (`[pkt_05-25_02-14_DN-014]`) to verify the underlying IntentPacket exists and the claim is supported. Or he clicks the "View provenance" link on the finding for the broader triplet view.

**Success Criteria:**
- Citation chip click reveals inline panel below the finding (not modal)
- Panel content: IntentPacket ID + arrival timestamp + train ID + payload type + 1-line payload excerpt + link to full packet detail in archive
- Multiple citation chips can be clicked → multiple inline panels stack below the finding
- "View provenance" link routes to Replay archive triplet detail (new tab) — full Provenance Bundle visible there (scenario 05.7 surface)
- Inline panel close affordance (×) per panel
- Esc closes all open citation inspections on the current brief
- Operator interaction logged per FR32

**Entry Points:**
- Click on citation chip in 13.2
- Click on "View provenance" link on a finding in 13.2
- Keyboard: Enter on focused citation chip

**Exit Points:**
- Click × on the inline panel → panel closes; brief returns to 13.2 state
- Click another citation chip → that chip's panel opens too (multi-panel stack)
- Click "Open packet in archive" inside the inline panel → opens new tab with full IntentPacket payload detail
- Click "View triplet in Replay archive" → opens new tab to scenario 05.7 surface with the brief's triplet preloaded
- Click outside any inline panel → no action (panels stay open until explicitly closed)
- Esc → closes all open citation inspections on current brief

## Components

### Citation Chip (in source finding — from 13.2)

| Element | Behaviour |
|---|---|
| Visual | Compact pill with packet ID in code style font (e.g. `[pkt_05-25_02-14_DN-014]`) |
| Hover | Tooltip preview (200ms delay): packet ID + train + 1-line excerpt — no full panel |
| Click | Opens inline panel below the finding (this sub-page's content) |
| Already-opened state | Chip shows subtle highlight indicating its panel is currently expanded; click again toggles closed |

### Inline Citation Panel

Structure per panel:

| Element | Content | Source |
|---|---|---|
| Header | "Citation: {packet_id}" + close button × | Static |
| Packet ID | Full identifier (no truncation) | Provenance Bundle |
| Arrival timestamp | "2026-05-25 02:14:23 (4h 16m ago)" — absolute + relative | IntentPacket arrival event |
| Train ID | "DN-014 (DOSTO Neu — 4729-014)" — internal + display reference | Cross-referenced |
| Payload type | "ALARM_RAISED" / "ALARM_CLEARED" / "HEALTH_CHECK" / etc. | Packet schema |
| Severity | HIGH / MEDIUM / LOW / CRITICAL (per packet, may differ from claim's severity badge) | Packet payload |
| Source | "brain MQTT → landside on oebb/DN-014/intent" — shows the data path | Inferred from packet metadata |
| Payload excerpt | Compact key-value display of the most relevant payload fields (4–6 lines typical) | Packet payload (truncated for inline display) |
| Action affordances | [Open packet in archive ↗] [View triplet in archive ↗] | Both open in new tab |

### Open Packet in Archive Affordance

| Element | Behaviour |
|---|---|
| Click | Opens new tab with full IntentPacket payload + raw packet view + arrival/storage metadata |
| URL | `/archive/intent-packets/{packet_id}` |
| Operator interaction | Logged per FR32 — `intent_packet_archive_opened_from_kondukt` with packet_id + brief recommendation_id context |

### View Triplet in Archive Affordance

| Element | Behaviour |
|---|---|
| Click | Opens new tab to scenario 05.7 Replay archive surface; the triplet that this brief produced is pre-loaded; this specific citation highlighted in the Provenance Bundle view |
| URL | `/archive/replay/{recommendation_id}?highlight={packet_id}` |
| Operator interaction | Logged per FR32 — `replay_archive_opened_from_kondukt` with recommendation_id + highlighted packet_id |

### "View provenance" link (from 13.2 — alternate entry path)

When Roland clicks "View provenance ↗" on a finding (rather than a specific citation chip), the behaviour differs slightly:

| Element | Behaviour |
|---|---|
| Click on finding-level "View provenance" | Opens Replay archive triplet detail in new tab — the broader view (full Provenance Bundle + Validation Receipt detail + all IntentPackets cited) |
| This is the same destination as "View triplet in archive" from inline panel | Both routes point to the same Replay archive surface; the finding-level shortcut skips the inline panel step |

### Validation Receipt Click (3-check per finding)

When Roland clicks a green check (✓ schema / ✓ citation-trace / ✓ advisory-language) on a finding:

| Element | Behaviour |
|---|---|
| Click on `✓ schema` | Hover-popover (or inline expansion) shows: which schema validated this output (skill_name + version) + which fields passed + validator rule references |
| Click on `✓ citation-trace` | Shows: which IntentPackets were checked against claims; which Provenance Bundle entries supported which claims; pass details |
| Click on `✓ advisory-language` | Shows: linter rules that ran (no imperative verbs, operator-as-subject, etc.); the LLM's draft passed all rules; rule references |
| Closed by default | These details are inspection-on-demand — not visible by default to keep the brief glanceable |

### Esc Behaviour

| Trigger | Effect |
|---|---|
| Esc key when any inline panel is open | Closes all open citation inspections on the current brief; brief returns to 13.2 default state |
| Esc when no panels open | No-op (or focuses skill registry — Phase 4 confirms) |

## State Variants

| State | Trigger | Visual |
|---|---|---|
| Default (panel open) | Roland clicked a citation chip | Inline panel below the finding; chip visually highlighted; close × available |
| Multiple panels open | Roland clicked multiple citation chips | Panels stack vertically below the finding; canvas scrolls if needed |
| Validator-rule popover open | Roland clicked a green check (✓ schema / citation-trace / advisory) | Small popover anchored to the check icon; auto-dismisses on click outside |
| Empty payload excerpt | Edge — packet payload is sparse | Excerpt shows "Payload: minimal; click 'Open packet in archive' for full detail" |
| Stale citation | Edge — citation references a packet that since became stale (per FR59) | Panel shows stale-data badge; "Last update {timestamp}" surfaces |
| Citation resolution failed | Hard edge — packet ID in citation can't be resolved against archive (shouldn't happen post-validator, but defence-in-depth) | Panel shows: "Citation resolution failed for {packet_id}. This should not occur. Please contact Nomad ops. Validation Receipt indicated pass — investigate." Records FR57 security event (validator-vs-runtime drift signal). |
| BYOK provider unavailable during inspection | Provider degraded between brief render and Roland's citation click | Inspection panel still works — it doesn't require LLM, just archive read. No degradation visible at this surface. |

## Behaviour Notes

- **Inline panels are not modal.** They expand the finding's vertical footprint within the canvas. Other findings remain readable. This preserves the "Roland is reading a brief" context.
- **Multi-panel stack is intentional.** Roland may want to compare three citations side-by-side (e.g. the three packets supporting the bus-controller pattern). All can be open simultaneously.
- **Citation-trace transparency.** Clicking ✓ citation-trace shows the validator's working — which claim in the LLM draft was supported by which packet. This is the surface that converts "harness says it passed" into "you can see why it passed."
- **Operator interaction tracking:**
  - `kondukt_citation_chip_clicked` (FR32) — engagement signal that Roland reads citations rather than skipping them
  - `kondukt_citation_chip_hovered` (Phase 1.5 telemetry — best-effort)
  - `kondukt_validation_check_clicked` (when Roland inspects a green check — engagement with validator transparency)
  - `intent_packet_archive_opened_from_kondukt`
  - `replay_archive_opened_from_kondukt`
  - `kondukt_provenance_inspected` — coarse-grained roll-up of any citation/triplet inspection (per-brief)
- **Performance:** citation chip click → inline panel render <100ms (data already cached from brief render). Open-in-archive new-tab opens within browser-standard time.
- **Accessibility:** chip + panel both keyboard-navigable; screen-reader announces "Citation pkt_05-25_02-14_DN-014, ALARM_RAISED, train DN-014" etc.

## Open Questions

- **Citation chip visual format** — `[pkt_05-25_02-14_DN-014]` working assumption (date-time-train compact). Operators may prefer `[02:14 DN-014]` (shorter). Phase 4 confirms with usability testing.
- **Payload excerpt depth** — 4–6 lines working assumption. Too few = Roland has to open archive; too many = inline panel too tall. Phase 4 confirms based on typical IntentPacket payload size.
- **Multi-panel stack scroll behaviour** — when 3 panels are open and extend below viewport, canvas scrolls. Should the brief content above scroll independently from the panels, or together? Working assumption: together (canvas is one scrollable region). Phase 4 confirms.
- **Validation-check click behaviour** — popover vs inline expansion. Working assumption: hover-popover for casual inspection; click for inline expansion (more substantial detail). Phase 4 confirms.
- **Citation-resolution-failure handling** — this should be impossible post-validator. If it occurs, it's a validator-vs-runtime drift signal (FR9 risk in FIA). Working assumption: surface the error + emit FR57 security event + log per FR32. Phase 4 confirms severity routing.
- **Open-in-archive new-tab vs side-panel** — currently spec'd as new tab to preserve Kondukt context. Alternative: side-panel split-view. Working assumption: new tab (consistent with 11.4 Kondukt annotation behaviour). Phase 4 confirms.
- **Replay archive deep-link with highlight** — URL param `?highlight={packet_id}` is the working assumption. Implementation depends on Replay archive surface design (scenario 05.7). Phase 4 cross-references.
