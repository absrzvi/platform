## Overview

**Page Purpose:** Confirm the DEFER was logged with Elena's IdP, reason, and expected-resolution-window. Confirm Klaus's brief reflects the deferral with full reason text. Auto-dismiss to card stack within 2.5s.

**User Situation:** Elena's IdP auth succeeded for the DEFER action; the row was written. She sees the confirmation; Klaus's brief now shows the camera firmware card with "Deferred by Elena Müller, reason: Schedule with next maintenance window."

**Success Criteria:**
- Confirmation visible within <500ms
- DEFER summary: vehicle + fault + reason category + expected window + Elena's IdP + timestamp
- Klaus's brief propagation confirmed
- Auto-dismiss after 2.5s; manual Done available
- Identical UX pattern to 04.6 — Elena learns one flow, applies it to both actions

**Entry Points:**
- Successful DEFER commit from [04.5-idp-auth-per-action](../04.5-idp-auth-per-action/04.5-idp-auth-per-action.md)

**Exit Points:**
- Auto-dismiss → [04.3-vehicle-card-stack-electrical](../04.3-vehicle-card-stack-electrical/04.3-vehicle-card-stack-electrical.md) with the camera firmware card now in DEFERRED state
- "Done" tap → same destination
- If both ACK + DEFER on the same vehicle are now complete (Elena's scenario goal): "Done" auto-dismiss → [04.2-electrical-vehicle-list](../04.2-electrical-vehicle-list/04.2-electrical-vehicle-list.md) (next vehicle)

## Technical Notes

- **DEFER commit response shape:** server returns `{audit_row_id, audit_row_short_id, idempotency_key, idp_display_name, recorded_at, reason_category, reason_free_text?, expected_resolution_window, brief_regeneration_status, vehicle_open_actions_count}`. The `vehicle_open_actions_count` field drives the scenario-completion indicator.
- **Auto-dismiss target depends on state:**
  - If `vehicle_open_actions_count == 0` → next destination = vehicle list (04.2)
  - Else → next destination = card stack (04.3) for the current vehicle
- **Undo within 30s:** if Elena spots a wrong reason on this confirmation, the "Undo" action files a supersedes_id row matching the DEFER row, with the supersedes-reason = "operator-correction-within-grace-window". After 30s, the undo affordance disappears; subsequent corrections use the modify-deferral flow (post-MVP).
- **Klaus's brief reflects the deferral:** the brief regeneration includes the reason category + free-text + Elena's IdP + expected-resolution-window, visible in [03.4-card-detail-warning](../../03-klaus-prearrival-shadow-signoff/03.4-card-detail-warning/03.4-card-detail-warning.md) under DEFERRAL DETAIL section.
