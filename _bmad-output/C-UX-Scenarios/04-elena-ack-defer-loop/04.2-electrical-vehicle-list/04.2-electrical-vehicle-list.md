## Overview

**Page Purpose:** Elena selected Electrical at the role selector. This page shows her the vehicles in active bays with electrical-domain open actions (ACK pending / DEFER pending), sorted by pending count + recency. She taps a vehicle to drill into its card stack.

**User Situation:** Elena finished fitting a part on Unit 4815; she wants to log the ACK + the camera firmware DEFER. She picks 4815 from the list.

**Success Criteria:**
- Vehicles with electrical-domain pending actions visible at top
- Each row shows vehicle + bay + pending action count (e.g., "Unit 4815 · Bay 5 · 2 actions pending")
- Large row touch targets for tablet use
- Visual indicator if a vehicle has a CRITICAL pending vs only WARNING
- Always-reachable "Switch role" affordance

**Entry Points:**
- From role selector (Electrical tile tapped)
- From back navigation after an ACK/DEFER completes

**Exit Points:**
- Tap a vehicle row → [04.3-vehicle-card-stack-electrical](../04.3-vehicle-card-stack-electrical/04.3-vehicle-card-stack-electrical.md)
- Tap "Switch role" → back to [04.1-role-selector](../04.1-role-selector/04.1-role-selector.md)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Role-context active, ≥1 vehicle with pending actions | Row list visible | Tap row, switch role |
| Empty | No vehicles with electrical-domain pending actions | "No open electrical actions right now. Tap Switch to change role." | Switch role |
| Loading | Initial load | Skeleton rows (3 placeholders) | None |
| Real-time update | New action arrives for a vehicle | Subtle toast "Unit {id} — new {severity} action"; affected row highlights briefly + reorders if severity changed | Tap row, dismiss toast |
| Stale (no refresh >5 min) | Server hasn't pushed an update in 5 min and tablet hasn't pulled | Subtle "Last updated {time} — refresh" indicator above list | Tap refresh |

## Technical Notes

- **Data fetched without IdP:** the role-filtered list is fetched with role context but no individual identity. The server returns vehicles in the bound depot for the role. IdP fires only when Elena ACKs/DEFERs (04.5).
- **Sort:** server-side per the rule above; client doesn't re-sort.
- **SSE updates:** the tablet subscribes to depot-wide electrical-action events while this page is active.
- **Real-time toast UX:** persona F1 — minimal disruption; toasts auto-dismiss after 4s, never block taps.
