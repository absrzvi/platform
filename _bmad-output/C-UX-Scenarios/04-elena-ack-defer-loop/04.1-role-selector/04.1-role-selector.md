## Overview

**Page Purpose:** Elena walks to the shared depot tablet mounted at Bay 5. She picks her role (Electrical) with one tap. The selector clears every 4 hours (multi-shift coverage) so the next technician doesn't inherit her role context. Identity is per-action (IdP fires when she ACKs/DEFERs), so role-share without identity-share.

**User Situation:** Elena finished fitting a replacement part on Unit 4815 and walks to the tablet. The tablet shows the role selector (because the 4-hour clear has elapsed since last selection, or because no role is currently active).

**Success Criteria:**
- 4 role touch-targets, each ≥120×120px, easily tappable with gloved hands
- One tap = role selected; no confirmation step
- Role persists for 4 hours; auto-clears at expiry; explicit "Switch role" affordance always reachable
- No identity assumed at this point — IdP fires only when Elena takes an action (per 04.5)
- Persona F1: friction tolerance is one tap per shift; we honour that

**Entry Points:**
- Tablet boot / wake from sleep with no active role
- "Switch role" tap from any 04.x screen
- Active role expired after 4 hours

**Exit Points:**
- Tap "Electrical" → [04.2-electrical-vehicle-list](../04.2-electrical-vehicle-list/04.2-electrical-vehicle-list.md)
- Tap "Mechanical" / "IT" / "ECM Manager" → analogous list (only Electrical detailed in this spec; siblings inherit pattern)

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Tablet boot / wake / 4-hour expiry | All four tiles equally prominent | Tap any tile |

## Technical Notes

- **Role context storage:** local-only (IndexedDB or sessionStorage); never sent to server unauthenticated. Server only sees role when an IdP-authenticated action carries it as a query/header param.
- **4-hour TTL:** measured from selection time; setInterval check every minute; at expiry, navigates to this page.
- **No identity required to select role:** the selector is a *filter*, not a *login*. Identity is verified per-action (04.5).
- **Tablet binding:** each shared tablet is bound to a depot + bay at provisioning time (offline-configurable). The top bar reads from this binding.
- **Touch targets:** WCAG 2.5.5 minimum 44×44 CSS px; this spec exceeds (120×120 min) for gloved-hand operation.
- **Persona F1 contract:** the 4-hour TTL is the upper bound (per the UX decisions doc). It is NOT shorter. A shorter TTL would force Elena to re-select more often, breaking adoption.
