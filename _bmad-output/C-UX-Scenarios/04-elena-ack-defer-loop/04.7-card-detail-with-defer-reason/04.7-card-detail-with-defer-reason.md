## Overview

**Page Purpose:** Elena drilled into the camera firmware WARNING card. The expanded view shows technician detail + the DEFER reason selector (dropdown of common reasons + free-text). She picks a reason, taps DEFER, and IdP auth fires (04.5) before the deferral commits. Klaus's brief surfaces the deferral with reason + Elena's IdP attached.

**User Situation:** Elena saw the camera firmware mismatch card. She decides to defer it to the next maintenance window — not service-impacting in current state. She picks the reason from the dropdown and taps DEFER.

**Success Criteria:**
- Full technician detail visible (cameras affected, current vs expected firmware version)
- Reason selector dropdown carries the locked taxonomy + free-text fallback
- Reason field is REQUIRED — DEFER button disabled until non-empty
- DEFER → IdP auth → write commits with reason + IdP attached
- Expected-resolution-window (next maintenance schedule) suggested where computable

**Entry Points:**
- Tap "DEFER" action button on a WARNING/INFO card in [04.3-vehicle-card-stack-electrical](../04.3-vehicle-card-stack-electrical/04.3-vehicle-card-stack-electrical.md)

## Technical Notes

- **Taxonomy LOCKED v0 (pre-pilot):** the 7-item list above is the v0 taxonomy. Pre-pilot blocker: ECM Manager (Roland) sign-off. Mary's Sprint 1 party-mode taxonomy (for modification reasons) is *different* — that's Stefan-facing; this one is Elena-facing.
- **Server-side expected-resolution-window:** server uses reason category + active maintenance schedule + fault severity to compute a suggested window. If no schedule data, falls back to "Next maintenance window" (generic). Elena can override.
- **Reason = "Other" required free-text:** schema-level constraint — "Other" without free-text is rejected. The UI surfaces this before commit.
- **DEFER row schema:** `{fault_id, vehicle_id, action_type: "DEFER", reason_category, free_text, expected_resolution_window, idp_subject, idp_issuer, recorded_at}`. Append-only same as ACK rows.
- **Klaus's brief regenerates:** the DEFER row triggers brief regeneration; Klaus's 03.4 view of the WARNING card surfaces the reason category, free-text, expected window, and Elena's IdP.
