## Overview

**Page Purpose:** Confirm the ACK was logged with Elena's IdP identity attached. Show the IdP-bound row reference + propagation status (ACK now visible in Klaus's brief). Auto-dismiss back to card stack within 2-3 seconds, or wait for Elena's tap. Sibling pattern to [03.7-signoff-success](../../03-klaus-prearrival-shadow-signoff/03.7-signoff-success/03.7-signoff-success.md).

**User Situation:** Elena's biometric / PIN succeeded; the ACK row was written with her IdP. She sees the brief confirmation and either taps Done or lets it auto-dismiss; the card stack returns with the door motor card now showing ACKED state.

**Success Criteria:**
- Confirmation visible within <500ms of action commit
- IdP identity ("Logged by Elena Müller") visible — Elena verifies it's her name on the row (persona F2 defensive)
- Short audit-row id + idempotency key visible with copy affordances
- Klaus's brief is informed (propagation indicator)
- Auto-dismiss after 2.5s; manual Done available

**Entry Points:**
- Successful ACK commit from [04.5-idp-auth-per-action](../04.5-idp-auth-per-action/04.5-idp-auth-per-action.md)

**Exit Points:**
- Auto-dismiss → [04.3-vehicle-card-stack-electrical](../04.3-vehicle-card-stack-electrical/04.3-vehicle-card-stack-electrical.md) with the door motor card now in ACKED state
- "Done" tap → same destination
- "View audit row" → opens [05.1-audit-export-view](../../05-renate-audit-export-review/05.1-audit-export-view/05.1-audit-export-view.md) for the row Elena just wrote (rare; defensive view)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Online success | ACK committed, brief regeneration completed | Standard view as specified | View audit, Done, auto-dismiss |
| Online success, brief regeneration async | ACK committed, but brief regen is queued | Propagation line shows "Propagating to Klaus's brief… (a few seconds)" with subtle pulse | View audit, Done, auto-dismiss |

## Technical Notes

- **ACK commit response shape:** server returns `{audit_row_id, audit_row_short_id, idempotency_key, idp_display_name, recorded_at, brief_regeneration_status: "complete"|"async"|"queued"}`. The client renders directly.
- **Brief propagation:** ACK row triggers brief regeneration for the affected vehicle; the regeneration job typically completes in <2s. If still in progress at this view, the propagation line updates live via SSE.
- **Multi-action chaining:** if Elena entered 04.6 from a flow where another action is pending on the same vehicle (e.g., the camera firmware DEFER), the auto-dismiss pauses and offers "Continue to next" — useful when Elena is wrapping up multiple cards in one bay visit.
- **Identity display:** the IdP display name comes from the IdP claim (`name` or `preferred_username` field). NEVER from a local cache. Confirms the identity attached to the audit row is the one in the IdP system.
