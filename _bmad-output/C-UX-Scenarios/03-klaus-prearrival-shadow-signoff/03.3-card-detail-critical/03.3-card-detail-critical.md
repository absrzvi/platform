## Overview

**Page Purpose:** Klaus drills into a CRITICAL fault card to see the resolution chain — the chronological story of detection → ACK → action → test result. ECM Manager lens (plain English, no fault codes). The page exists because Klaus's persona W1 wants the *chain*, not just the summary, before walking to the bay.

**User Situation:** Klaus saw the CRITICAL card on the brief (03.2). He taps it to see what happened, who handled it, when.

**Success Criteria:**
- Resolution chain renders as a vertical timeline (detection → conductor ACK → technician assignment → part fitted → test result)
- Each event shows actor IdP, timestamp, brief plain-English description
- ECM Manager lens — no technical detail (fault codes, ports, packet traces hidden); link to technician-depth view if Klaus wants
- Collapse returns Klaus to the brief feed without losing scroll position

**Entry Points:**
- Tap on CRITICAL card in [03.2-prearrival-brief](../03.2-prearrival-brief/03.2-prearrival-brief.md)

**Exit Points:**
- Collapse → returns to brief feed at same scroll position
- Tap "View technician detail" → switches lens to technician view (out of MVP — disabled link with tooltip)
- Tap "Record sign-off" (header CTA) → [03.5-return-record-entry](../03.5-return-record-entry/03.5-return-record-entry.md)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Expanded (default after tap) | User tapped card | Full expansion visible; smooth 300ms height transition | Collapse, navigate |
| Single-card-expanded enforced | User taps a different card while this one is expanded | This card collapses (200ms); new card expands (300ms); single-expansion rule | n/a |
| Loading detail | Resolution chain still fetching after initial card render | Card body shows shimmer placeholder for chain section | None until loaded |
| Incomplete resolution | Fault has events but no resolution yet | Last timeline event has a pulsing dot; status line shows "Status: Resolution incomplete — awaiting {next_step}" | Collapse, navigate |
| Multi-actor cross-domain fault | Fault tagged for multiple roles (e.g., Electrical + IT) | Timeline shows all actors regardless of role; Klaus sees cross-domain participation | Collapse, navigate |

## Technical Notes

- **Chain fetched on expansion:** the brief load returns only the card-level fields; the resolution chain fetches on expansion (`GET /api/fault/{fault_id}/chain`). Saves bandwidth on a mobile PWA.
- **Single-card-expanded:** managed at the feed-level state. Tapping a new card collapses the current.
- **Lens switching (post-MVP):** the View Technician Detail link is wired to the same fault id with a `?role=technician` parameter (or analogous), opening Elena's view. Disabled in MVP.
- **Timeline accessibility:** the ol is semantically a chronological list; dot indicators have aria-hidden=true since the timestamp + event title carry the meaning textually.
- **Plain-English generation:** the description field is generated server-side per role at brief-generation time (same pipeline as the card summary).
