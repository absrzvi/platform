## Overview

**Page Purpose:** Confirm the shadow row write succeeded; surface the idempotency key (for traceability) and the audit row's short-id; transition the brief card visually from "ready for sign-off" to "cleared (shadow matched)". Klaus pockets his phone.

**User Situation:** Klaus held through the 5s countdown in 03.6; the atomic write fired and succeeded. He sees this success view briefly before being returned to the brief, which now shows Unit 4722's card in its "cleared" terminal state.

**Success Criteria:**
- Confirmation visible within <500ms of atomic write completion
- Idempotency key + short audit-row-id displayed for traceability (Klaus can reference them later if audit asks)
- Modal auto-dismisses to the brief view within 3 seconds (with manual "Done" button available)
- The brief card for Unit 4722 transitions visually to "cleared" state (same brief view that 03.2 lives in, but with this vehicle's card now in terminal state)
- Offline-queued state surfaces correctly when applicable

**Entry Points:**
- Successful atomic write from [03.6-hold-to-record](../03.6-hold-to-record/03.6-hold-to-record.md)

**Exit Points:**
- Auto-dismiss after 3s → returns to brief view with the cleared card
- Manual "Done" tap → same destination
- Manual "View audit row" tap → opens [05.1-audit-export-view](../../05-renate-audit-export-review/05.1-audit-export-view/05.1-audit-export-view.md) for the just-written row (Klaus can verify his own sign-off was recorded correctly)

## Technical Notes

- **Atomic write response payload:** the `POST /api/audit/shadow-events` response includes `{audit_row_id, audit_row_short_id, idempotency_key, recorded_at}`. The client renders these directly.
- **Auto-dismiss timer:** 3000ms; configurable per deployment via app config. Default chosen so Klaus has time to read but isn't forced to wait. Auto-dismiss is paused if the user is actively interacting with the metadata block (e.g., copying a key).
- **Brief card transition:** when this view auto-dismisses, the brief view re-renders Unit 4722's card in its cleared state ("Cleared (shadow matched at {time})"). The brief view's data refetch is triggered by the dismiss event.
- **Offline-queued state visual treatment:** the local UUID is shown in the same monospace style as the audit-row-id, but with a clock icon prefix to communicate "not yet synced." On sync, the local UUID is replaced with the server-issued short_id in the brief card's badge.
- **Haptic on success:** subtle iOS notification-success / Android equivalent haptic when the success view loads. Single tick, not pattern. Matches persona F2 — confirmation, not celebration.
