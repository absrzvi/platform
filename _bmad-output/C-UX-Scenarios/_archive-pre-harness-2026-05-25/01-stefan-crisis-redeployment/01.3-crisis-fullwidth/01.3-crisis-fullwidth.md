## Overview

**Page Purpose:** Full-width crisis-handling surface for a single active incident. Kondukt chat panel pre-seeded with incident context (resolution chain summary + replacement unit hint). The chat panel is the operator's primary dialogue interface; chat composer below for follow-up queries.

**User Situation:** Stefan tapped "View" on the crisis modal. He needs context fast — what happened, who already touched it, what's available — before he forms a query.

**Success Criteria:**
- Initial chat seed renders within 1.5s of page entry
- Pre-seeded message contains: detection timestamp, conductor ACK status, ECM clearance status, replacement unit suggestion (if applicable)
- Chat composer focused and ready for typing

**Entry Points:**
- From [01.2-crisis-modal](../01.2-crisis-modal/01.2-crisis-modal.md) "View" tap (carrying `intent_id` query param)
- Direct URL: `/crisis/{intent_id}`

**Exit Points:**
- Stefan submits free-form query → [01.4-tool-call-render](../01.4-tool-call-render/01.4-tool-call-render.md)
- Stefan exits crisis without acting → "Back to triage" returns to [01.1-triage-view](../01.1-triage-view/01.1-triage-view.md)
- Kondukt becomes unavailable mid-session → degraded variant (scenario 10.3)

## Layout Structure

Full-width single-column conversation surface. Top header strip with incident metadata. Chat history scrolls. Composer fixed to bottom.

```
+----------------------------------------------------------+
| INCIDENT HEADER STRIP                                    |
|  Unit 4722 | Door Motor Car 4 | CRITICAL | ← Triage      |
+----------------------------------------------------------+
|                                                          |
|  CHAT HISTORY (scrollable, fills viewport)               |
|  ┌────────────────────────────────────────────────────┐  |
|  │ Kondukt avatar │ Pre-seeded context message       │  |
|  │                │ "Door motor fault detected 06:14.│  |
|  │                │  Conductor resolved manually...  │  |
|  │                │  Replacement unit available: 4819│  |
|  │                │  [confidence pill: HIGH]"        │  |
|  └────────────────────────────────────────────────────┘  |
|                                                          |
|  (subsequent messages render here as Stefan converses)   |
|                                                          |
+----------------------------------------------------------+
| COMPOSER                                                 |
|  [textarea: "Ask a follow-up..."]            [Send →]    |
+----------------------------------------------------------+
```

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Initial load | Page entered, awaiting seed | Header rendered, chat area shows shimmer placeholder ("Loading incident context…"), composer disabled | None |
| Seeded | Pre-seeded message rendered | Header + seed message visible, composer focused & enabled | Type, send, back |
| Awaiting response | After Stefan sends a query | User's message appears in chat; pending indicator below (typing dots); composer remains usable | Send another, back |
| Stale seed | IntentPacket update arrives mid-session | Toast surfaces: "New information for Unit 4722 — refresh?"; Stefan opts in to re-seed | Refresh, ignore |
| Kondukt unavailable mid-session | 5+ min sustained 429/5xx | Degraded banner appears in header; composer disabled; "Use direct tool call instead" affordance appears → see scenario 10.3 | Direct tool, back |

## Technical Notes

- **Seed generation:** server-side. The endpoint `/api/crisis/{intent_id}/seed` returns a structured response: detection timestamp, ACK chain, ECM clearance, replacement unit candidate(s), confidence. The client renders deterministically — no client-side LLM call for the seed.
- **Subsequent chat turns:** routed through the Kondukt agent runtime (per Option A — direct Anthropic SDK + MCP client per design-log decision 2026-05-25). Tool calls render inline (see 01.4).
- **Chat history persistence:** on page exit/back navigation, history persists for the lifetime of the `intent_id`. Multi-shift handover surfaces the chat history (scenario 09).
- **SSE stream:** Kondukt response streamed token-by-token; tool-call pills render as structured events within the stream.
