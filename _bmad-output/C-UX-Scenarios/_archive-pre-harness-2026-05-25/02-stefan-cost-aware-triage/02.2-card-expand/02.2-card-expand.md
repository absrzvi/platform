## Overview

**Page Purpose:** When Stefan clicks a card in the triage feed (non-crisis), it expands inline to reveal: fault detail, full cost-impact summary, source data (IntentPackets), and a "Generate recommendation" affordance. The expanded card is the **bridge between scanning and active engagement** — Stefan has bandwidth to investigate but hasn't committed to a recommendation request yet.

**User Situation:** Stefan clicked the WARNING card for Unit 4831 brake wear. He wants enough context to decide whether to ask Kondukt for a recommendation, without leaving the triage feed.

**Success Criteria:**
- Expansion happens in <300ms with smooth height transition
- Other cards in the feed remain visible (no modal blocking)
- Full cost-impact summary shows source data attribution (which IntentPacket, which historical baseline)
- "Generate recommendation" affordance is the primary action; collapse is secondary

**Entry Points:**
- Card click in [02.1-triage-view](../02.1-triage-view/02.1-triage-view.md)

**Exit Points:**
- "Generate recommendation" → [02.3-rao-card](../02.3-rao-card/02.3-rao-card.md) (RAO renders inline at expanded card position)
- "Collapse" or click outside → returns card to collapsed state in feed (same 02.1 view)
- Stefan clicks a different card → that card expands, current collapses

## Layout Structure

The clicked card expands in place. Its height grows; cards below shift down (with smooth animation). Inside the expanded region, three stacked sub-sections: fault detail, full cost-impact, source data.

```
┌── Expanded card (was collapsed; now full height) ──────┐
│  ▶  [severity band] │ Header (system label + timestamp)│
│                     │ Summary (1–2 lines)               │
│                     │ Cost-impact one-liner             │
│  ─────────────────────────────────────────────────────  │
│  FAULT DETAIL                                           │
│    Plain English: …                                     │
│    Technical view: …  (collapsed by default)            │
│  ─────────────────────────────────────────────────────  │
│  COST IMPACT — full                                     │
│    Maintenance window: 4h                               │
│    Estimated cost: €380                                 │
│    Delay impact if not scheduled: ~12 min/service       │
│      over 2 weeks of affected rotations                 │
│    Source data: IntentPacket {id} · HAFAS rotation set  │
│  ─────────────────────────────────────────────────────  │
│  [ Collapse ]              [ Generate recommendation ]  │
└────────────────────────────────────────────────────────┘
```

## Technical Notes

- **Expansion mechanism:** CSS `height` transition (300ms ease-out); virtualised list adapts to changed card height via `@tanstack/react-virtual` measurement callback.
- **Single-card expansion:** managed at the feed-level state (Zustand `expandedCardId` or React Query selected card). Clicking a different card collapses current.
- **Source data drawer:** right-side panel, max-width 480px, slides in from right. Shows the raw IntentPacket JSON, the HAFAS rotation data, and the cost-impact computation steps. Copy-to-clipboard for each block.
- **Generate recommendation:** dispatches a Kondukt agent run with the card's `intent_id` as context. RAO arrives streamed; the expanded card's bottom region transitions from "Generate recommendation" button → RAO card surface (see 02.3).
- **Keyboard navigation:** Tab order = severity → header → fault detail toggle → tech detail (if open) → cost fields → action buttons.
